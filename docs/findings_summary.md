# Сводка наблюдений: GNN vs XGBoost на IBM AML, и где граф реально полезен

Задача: edge-classification на направленном мультиграфе «узел = счёт, ребро = транзакция»,
IBM AML HI-Small (515k узлов, 5.08M рёбер, illicit ≈ 0.10%). Метрики при дисбалансе —
AUC-PR / F1-minority / ROC-AUC (accuracy не отчитываем); порог фиксируется по val, метрики —
на полном test. Все числа — `results/*.json`, рисунки — `results/*.png`.

---

## 1. Главный результат: граф полезен как ПРЕДСТАВЛЕНИЕ (гибрид GNN→XGBoost)

Эмбеддинг ребра из обученной GNN (`[h_u ‖ h_v ‖ e_label]`, 192-мерный) подаётся
дополнительными признаками в XGBoost. Это приём Inspection-L (GNN-фичи + бустинг).

| модель | test AUC-PR | F1 | ROC-AUC |
|---|---|---|---|
| лучший чистый GNN (PNA, Kaggle) | 0.059 | 0.087 | 0.964 |
| XGBoost (только сырьё ребра) | 0.288 | 0.397 | 0.963 |
| **Гибрид + эмбеддинг base GINe** | **0.310** | 0.403 | **0.986** |
| **Гибрид + эмбеддинг Multi-GNN (rev+port+ego, Kaggle)** | **0.330** | 0.398 | 0.958 |

**Граф несёт сигнал, которого нет в атрибутах транзакции:** добавление графового
эмбеддинга поднимает raw-edge табличный детектор 0.289 → 0.330 (+4.1 п.п. AUC-PR).
**Прирост монотонно растёт со структурностью** эмбеддинга (base GINe +2.2 п.п. →
Multi-GNN +5.9 п.п.) — значит вклад идёт именно от графовой топологии, а не от
дублирования табличных признаков.

Анти-утечка соблюдена: эмбеддинги извлекаются на тех же контекстах, что и обучение GNN
(train-контекст для train/val-рёбер, train+val для test); XGBoost учится на train,
меряется на полном test. Воспроизведение: `python scripts/hybrid_gnn_xgb.py - checkpoints/ibm_multignn_fulldata.pt`.
Артефакт: `results/ibm_hybrid_gnn_xgb_metrics.json`. Текущий сохраненный JSON —
полный Kaggle-прогон с precision/recall/threshold.

**Снятый парадокс:** Multi-GNN как *самостоятельный* классификатор хуже base GINe
(0.041 < 0.054), но её *эмбеддинг* богаче структурой и в гибриде помогает СИЛЬНЕЕ.
Адаптации reverse/port/ego не бесполезны — они обогащают представление, просто не
для standalone-классификации при 0.1% дисбалансе.

---

## 2. Почему чистая GNN проигрывает: сигнал HI-Small — табличный

Feature-ablation XGBoost (тот же протокол, разный набор фич):

| фичи XGBoost | test AUC-PR | F1 |
|---|---|---|
| только сырьё ребра (amount/currency/format + parallel) | **0.288** | 0.397 |
| полный (сырьё + узловые degree/sum) | 0.240 | 0.298 |
| только графовые узловые (degree/sum) | 0.067 | 0.140 |
| для сравнения: лучший чистый GNN (Kaggle PNA) | 0.059 | 0.087 |

Вывод: **метку определяют атрибуты транзакции**, а не топология. Сырьё-XGBoost (0.288)
≫ структура-only (0.067 ≈ уровень чистого GNN). Узловые графовые фичи дереву даже
слегка вредят (шум). Поэтому чистая GNN, чья сила — структура, упирается в слабый
структурный сигнал; а градиентный бустинг выжимает табличный сигнал кратно лучше
нейросети (Grinsztajn 2022: trees > DL на табличке). XGBoost-число НЕ завышено —
наоборот, без графовых узловых фич он ещё сильнее.

## 3. RQ2: адаптации Egressy (reverse/port/ego) и edge-updates не переносятся

Полный ablation в трёх режимах обучения (test AUC-PR, base GINe / Multi-GNN full):
with-time 0.019/0.051; no-time 0.044/0.013; **full-data/no-time 0.054/0.041**.
Устойчивого переноса эффекта адаптаций нет ни в одном корректном режиме; в with-time
они «помогали» — артефакт компенсации вреда norm_time. norm_time при temporal split
вреден (тестовые времена вне train-диапазона).

**Лестница «слабый → сильный режим» L0→L6** (full-data/no-time, фиксированный протокол,
по одному рычагу за шаг) — `results/ibm_ladder.md`, `results/transition_ladder.png`:

| шаг | модель | test AUC-PR |
|---|---|---|
| L0 | GINe (base, [10,10]/2) | 0.054 |
| L1 | Multi-GNN (rev+port+ego) | 0.041 |
| L2 | + edge-updates | 0.035 |
| L3 | Multi-GNN+EU, большие окрестности [50,50,50]/3 | 0.042 |
| **L4** | **чистый PNA** ([50,50,50]/3, Kaggle batch 2048) | **0.059** |
| L5 | Multi-PNA (rev+port+ego) | 0.057 |
| L6 | Multi-PNA+EU (целевой SOTA-рецепт) | 0.040 |

PNA остается лучшим подтвержденным standalone GNN в Kaggle verification, но порядок
GINe/Multi-GNN/PNA чувствителен к sampling/batch: в Kaggle Multi-GNN выше GINe, а
PNA выше обоих, но все они далеко ниже XGBoost/hybrid. ROC-AUC GNN высокий (до 0.964) — сигнал и
ранжирование есть, проигрыш именно в precision при 0.1% prevalence.

---

## 4. RQ3 (per-pattern) и эвристики

XGBoost лидирует по recall на всех 8 паттернах (включая структурные: cycle, scatter-gather).
Степенные эвристики не работают: illicit-счета НИЖЕ по степени, чем легитимные
(ROC-AUC degree-эвристики ≈ 0.22) — отмывание идёт через низкостепенных «мулов», не хабы.
Это структурная диагностика, объясняющая, почему наивная топология слаба.

---

## 5. Контекст литературы (выигрыш GNN узкий и хрупкий)

У Egressy (AAAI 2024) на AML Small HI GBT обходят только 3 конфигурации GNN
(Multi-PNA+EU 68.2, Multi-PNA 64.6, Multi-GIN+EU 64.8 vs GBT+GFP 63.2), и то на 1–5 п.п.;
всё проще (GIN 28.7, Multi-GIN 57.2) проигрывает дереву. Их GBT — это GBT+GFP (богатые
подграфовые фичи) + тюнинг, поэтому их абсолютные числа выше и для дерева, и для GNN.
Несколько работ (Blanuša, Deprez, Lorenz, Grinsztajn) находят обратное: деревья ≥ GNN на
финансах/табличке. Наш результат попадает в эту ветку и не противоречит Egressy (его
выигрыш тонкий и требует полного рецепта, который мы не воспроизводили дословно).

---

## Итоговый тезис (защищаемый)

Чистая GNN на edge-классификации AML проигрывает градиентному бустингу, потому что сигнал
HI-Small преимущественно **атрибутивный** (доказано feature-ablation), а не топологический.
Однако графовое представление несёт **комплементарный структурный сигнал**: эмбеддинг GNN,
поданный фичами в XGBoost, повышает AUC-PR 0.289 → 0.330. Таким образом, граф полезен как **представление в гибридной схеме**,
даже когда уступает как самостоятельный классификатор. Это и есть граница применимости
графов для данной задачи — позитивный, измеримый и воспроизводимый вклад.

Оговорки (честно): один сид (42); эмбеддинги из чекпоинтов слабого режима ([10,10]/2 слоя);
Kaggle verification выполнен для одного seed; L5/L6 PNA summary
восстановлены из run log без per-pattern; faithful-репродукция Multi-PNA+EU
(раздельные AGG_in/out) и GFP-честное дерево — вне объёма. Manifest финальных
артефактов: `docs/final_results_manifest.md`.
